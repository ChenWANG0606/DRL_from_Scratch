# 深度强化学习

1. 深度强化学习：不使用表格记录具体值，而是将策略给参数化
   - 基于策略的学习：策略梯度
   - 基于价值的学习：Q学习

## Policy Gradient

1. 策略梯度：假设参数为$\theta$的策略为$\pi_\theta$，该策略为随机性策略，其输入为某个状态，输出一个动作的概率分布
   - 基于策略的方法，对策略进行了参数化
   - 策略梯度不需要再动作空间中最大化价值，因此较为适合解决具有高维或者连续动作空间的问题

## 策略梯度算法-强化学习为例

1. 强化学习（Reinforce）算法有三个组成部分，演员、环境和奖励函数

   - 这里是Reinforce专有的算法名称，不是统称意义上的强化学习
   - 以视频游戏为例

   - 演员：演员是负责操控游戏的摇杆，可以选择各种操作
   - 环境：游戏的主机，负责控制游戏的画面、怪兽的移动
   - 奖励函数：当我们做某件事情或者发生某个状况时，可以得到的分数（击杀怪兽）

2. 强化学习中环境环境和奖励函数是给定的，我们要做的事调整演员的策略，即给定一个输入，输出演员现在要执行的策略

   ![image-20260222160734147](assets/image-20260222160734147.png)

3. 定义一个深度强化学习问题：		![image-20260222161518142](assets/image-20260222161518142.png)

   - 策略是一个网络：

     - 对于策略$\pi$，$\theta$是策略网络的参数
     - 网络的输入是智能体能看到的东西
       - 比如游戏画面
     - 网络的输出是要执行的动作
       - 对于离散的动作可以有几个动作就有几个神经元
         - 每个神经元代表一种动作
         - 每个神经元输出的分数是一个概率，对应每种动作执行的**概率**

   - 状态：初始游戏画面可以记为$s_1$...

   - 动作：第一次执行的动作可以记为$a_1$...

   - 回合：一场游戏称为一个回合

   - 环境：可以视为一个函数，输入一个动作，输出新的状态（也可以是一个概率）

   - 轨迹：一场游戏中，环境输出的s与演员动作a全部组合起来，称为一个轨迹

     - $$\tau = \{s_1, a_1,s_2, a_2, ...s_t,a_t\}$$

     - 给定演员的参数$\theta$，可以计算某种轨迹发生的概率

       ![image-20260222161952515](assets/image-20260222161952515.png)

   - 奖励：可以视为一个函数，输入$s_i, a_i$，输出$r_i$
     - 在$s_1$执行$a_1$得到的奖励记为$r_1$...
   - 总奖励：将一场游戏的总奖励加和得到就是总奖励，对于一个轨迹$\tau$，总奖励也称为回报，可以记为$R(\tau)$

4. 策略梯度

   - 演员要想办法最大化它可以得到的奖励：在某一场游戏的某一个回合里，可以得到$R(\tau)$，我们要做的事调整演员内部参数$\theta$，使得$R(\tau)$越大越好
     - 对于参数$\theta$，策略$\tau$，$R(\tau)$是一个随机变量，我们可以计算它的期望
       - $$\bar{R_\theta} = \sum_{\tau}R(\tau)p_\theta(\tau)$$

6. 策略梯度与分类问题对比

   - 以手写体分类为例，输入图片，输出各种类别概率，目标是输出的概率分布逼近真实概率分布

   - 交叉熵：可以表示两个概率分布的差距

     - 对于二分类问题，交叉熵就是二项分布的负对数似然

     - 从信息论角度：用模型分布 q 来编码真实分布 p 时的平均编码长度

       - 真实世界按 p 产生数据
       - 你却用模型 q 去预测
       - 交叉熵衡量：你因为预测错误而多付出的信息代价

     - 分布中：$H(p,q) = H(p) + KL(p||q)$

       - 交叉熵：世界本身的不确定性，加上模型的错误
         - $H(p,q) \;=\; -\sum_{x\in\mathcal{X}} p(x)\,\log q(x)$
       - H(p)是香农熵:表示Hp是最优编码下，每个符号所需要的最少平均比特数
         - $H(p) \;=\; -\sum_{x\in\mathcal{X}} p(x)\,\log p(x)$
         - 衡量了本身的不可预测性，不确定性、随机性、混乱程度、可压缩性
           - 接近均匀分布越随机，不确定、混乱，不可压缩
           - 某个概率极端不均匀$p = (0.99,0.11)$：几乎总是同一个结果，不需要太多信息
         - 给定真实分布p，Hp是固定的

       - KL散度：当真实分布是 p，但你用 q 来描述世界时，平均多付出的信息代价。
         - $KL(p\|q) \;=\; \sum_{x\in\mathcal{X}} p(x)\,\log\frac{p(x)}{q(x)}$
         - 是一个似然比，衡量了p比q的优势
         - 不对称，不是距离

   - 策略梯度预测的是每个动作的概率，由于没有真实标签，我们使用采样到的动作作为标签，处于补偿，我们用奖励汇报Gt（该动作未来回报的折现）来表示该动作的评价，

### 策略梯度实现技巧

1. 添加基线：

   - 一个问题：如果给定状态s采取动作a，整场游戏都得到正的奖励，那么就要增加(s,a)的概率，如果同样情况下正常游戏都是负的奖励，那么就要减小(s,a)的概率

     - 但是如果奖励总是正的呢？

     - 理想情况下

       - 由于R都是正的，模型会被要求提升所有可能动作的概率，R高的增加的多，R少的增加的少

       - 概率和为1，对数概率和为0，提高少（b）的在做完归一化后，概率是下降的，提高多的概率是上升的（a、c）

         ![image-20260223153557120](assets/image-20260223153557120.png)

     - 实际情况下

       - 使用采样来计算期望：对所有可能得s和a的对进行求和
       - 但是真正学习的时候只采样了少量s和a的对，可能很多动作从来都没有被采样到
       - 如下图，a动作从来没被采样到，因此a的概率下降，b和c的概率提高
         - 产生了偏差，a下降并不是因为他不好

   - 方案：让奖励并不总是正的

     - 给奖励设置一个基线b，将所有奖励减去b

       ![image-20260223154013878](assets/image-20260223154013878.png)

     - 基线b的设计：对$\tau$取期望，计算平均值，令$b \approx E[R(\tau)]$
       - 在训练时，通过不断把$R(\tau)$记录下来，动态计算其平均值

2. 分配合适的分数

   - 原本的公式中，同一场游戏里所有状态动作对都是用同样的奖励进行加权

     - ![image-20260223154406467](assets/image-20260223154406467.png)

   - 问题：同一场游戏中有些动作是好的，有些动作是坏的，整场游戏的结果是好的不代表每一个动作都是好的，反之亦然

     - 理想状况下，采样足够多的时候，各种因素抵消，总的奖励可以反应其中动作好坏，但是采样不足的时候，这是没有代表性的

   - 解决方法：

     - 一个做法是不要把正常游戏的奖励全部加起来，而是只计算这个动作之后的到的奖励

       - 因为动作发生前得到的奖励和这个动作是无关的

         ![image-20260223155816629](assets/image-20260223155816629.png)

     - 更进一步，可以对未来奖励计算折扣

3. 优势函数$A^\theta(s_t,a_t) = \sum_{t' = t}^{T_n}r_{t'}^n-b$：结合上面两个技巧

   - b可以是依赖状态的，由网络进行估计，R-b称为优势函数
   - 未来期望的加总需要一个模型和环境进行交互，才能知道接下来的奖励
   - 优势函数上的$\theta$代表用来与环境交互的模型$\theta$
   - 优势函数的意义：假设在某一个状态st执行一个动作at，相较于其他动作，at有多好
     - 相对优势：不在乎绝对的好，而是相对的好，基线b就是这个动作相对好的程度
   - 优势函数如果使用网络估计，这个网络就成为评论员

### Reinforce算法 蒙特卡洛梯度

1. 蒙特卡洛和时序差分

   - 蒙特卡洛方法：完成一个回合，再利用回合的数据进行学习，做更新

     - 因为得到了整个回合的数据，因此可以获得每一步的奖励，也可以计算每个步骤未来的奖励总和

   - 时序差分：每个步骤更新一次

     - 更新频率更高，时序差分方法的Q函数可以近似未来总奖励$G_t$

       ![image-20260223162312930](assets/image-20260223162312930.png)

2. REINFORCE：策略梯度的最经典的算法

   - 更新方式：回合更新

     - 先获得每个步骤的奖励，然后计算每个步骤未来的总奖励Gt，然后代入策略梯度公式

       - 即上一个步骤和下一个步骤的未来总奖励的关系，在代码上是reverse然后递推向前

## 深度Q网络算法

1. 传统强化学习的问题：使用表格记录状态价值函数或动作价值函数
   - 连续空间，无穷多状态，无法使用表格
   - 方法：使用价值函数近似，利用函数直接拟合状态价值函数或者动作价值函数，降低对存储空间的要求
2. 价值函数近似：在连续的状态和动作空间中计算值函数，可以用一个函数来近似计算，成为价值函数近似
   - $$Q_\phi(s,a) \approx Q_{\pi}(s,a)$$
   - 其中s，a分别是状态s和动作a的向量表示，函数通常是一个参数为$\phi$的函数，比如神经网络，其输出一个实数，成为Q网络
3. 深度Q网络：基于深度学习的Q学习方法，结合了价值函数近似于神经网络技术，采用目标网络和经验回放的的方法来进行网络的训练
   - 深度Q网络：基于价值的算法，学习的不是策略，而是评论员
4. 演员：学习一个策略来得到尽量高的回报
5. 评论员：评价（演员）动作的好坏
   - 实际上做的事策略评估
   - 两种评论员
     - 状态价值函数评论员$V_\pi$
     - 动作价值函数评论员Q函数

### 状态价值函数

1. 状态价值函数评论员$V_\pi$​：假设演员的策略是$\pi$，使用$\pi$和环境交互，当智能体看到状态s时，接下来的累计奖励的期望值是多少

   - 接受s输出一个标量

   - 问题：评论员无法凭空判断状态的好坏，他评价的是给定某一个状态的时候，接下来交互的演员的策略是$\pi$，我们会得到的奖励是多少

     - 同样的状态，策略不一样，得到的奖励也不一样

     - 评论员需要绑定一个演员，评论员的输出值取决于状态和演员，它在衡量某一个演员的好坏而不是状态的好坏
     - 评论的输出是和演员有关的，状态的价值实际取决于演员，当演员改变状态价值函数的输出其实也会跟着改变

2. 状态价值函数的计算：

   - 蒙特卡洛：

     - 让演员和环境交互，看演员好不好就让演员和环境交互，让评论员评价

     - 评论员统计演员如果看到某个状态s，接下来的累计奖励有多大

     - 然后用状态价值函数近似网络进行估计，由于网络有泛化能力，在有限的观察中尽量学出全部规律

       - 回归问题：输入状态s，输出价值G

       ![img_v3_02v7_1a154d73-27e4-4741-9c92-658061446c2g](assets/img_v3_02v7_1a154d73-27e4-4741-9c92-658061446c2g.jpg)

   - 时序差分：在某状态$s_t$，采取动作$a_t$等到奖励$r_t$，接下来进入状态$s_{t+1}$

     - $V_\pi(s_t) = V_\pi(s_{t+1})+r_t$

     - 不直接估计$V_\pi$，而是希望得到的结果$V_\pi$可以满足上式子

       - 输入$s_t$得到$V_\pi(s_t)$
       - 输入$s_{t+1}$得到$V_\pi(s_{t+1})$
       - $V_\pi(s_{t+1})$与$V_\pi(s_t)$的差应该是$r_t$ 希望损失的详见和奖励接近

       ![img_v3_02v7_778490c0-6bde-43c0-a89e-30270cf237dg](assets/img_v3_02v7_778490c0-6bde-43c0-a89e-30270cf237dg.jpg)

   - 蒙特卡洛vs时序差分

     - 方差区别

       - 需要再次说明的是，蒙特卡洛方差更大
         - 因为游戏和模型本身都有随机性
         - $G_a$是多步奖励的和，$Var[kX] = k^2Var[X]$，通过多步骤的累计，相比于某一个状态的奖励，方差是很大的
       - 时序差分的目标是最小化$Var_\pi(s_t) <-->r+V+\pi(s_{t+1})$
         - r具有随机性，即便在s采取同一个动作，奖励也不一定都是一样的
         - G是多步r的求和，因此方差更大
       - 问题：$V_\pi$不一定准确，假设$V_\pi$不准确，使用上面目标学出来的结果也不准确
         - 因此蒙特卡洛和时序差分各有优劣
         - 时序差分更常用，蒙特卡洛方法比较少

     - 计算结果区别

       ![img_v3_02v7_6af76f61-ab46-462d-ade0-25bbf3f40c7g](assets/img_v3_02v7_6af76f61-ab46-462d-ade0-25bbf3f40c7g.jpg)

       - 场景：假设某个评论员，它去观察某一个策略$\pi$与环境交互8个回合的结果

         - 有一个有sa
         - 剩下都是sb

       - sb的计算：8场游戏，sb都存在，计算看到sb后得到的奖励，期望是3/4

       - sa的计算：

         - 蒙特卡洛：
           - sa出现一次，看到sa接下来的累计奖励就是0，所以sa的期望奖励就是0
         - 时序差分：
           - 更新$Var_\pi(s_a) = r+V+\pi(s_b)$
           - 在状态a得到奖励0之后，进入状态sb，sa的奖励等于状态sb的期望奖励加上从状态sa进入状态sb的期望奖励r
             - r = 0
             - sb = 3/4
             - sa的奖励是3/4

       - 为啥不一样？sa这个时间的存在有两种可能

         ![img_v3_02v7_18ac6696-cfb8-465c-b1c5-9a1e46c9e4fg](assets/img_v3_02v7_18ac6696-cfb8-465c-b1c5-9a1e46c9e4fg.jpg)

### 动作价值函数

1. 动作状态值函数评论员Q函数：输入一个状态-动作对，即在某一个状态采取某一个动作，假设都是用策略$\pi$，得到的累计奖励的期望值有多大
   - 状态价值函数输入一个状态，根据状态计算出这个状态以后的期望的累计奖励是多少

2. Q函数要注意的问题
   - 策略在看到状态s时，不一定采取动作a
   - Q函数强制状态s采取动作a，而不管策略会不会采取a
     - 这并不重要
   - 在某个状态强制执行一次动作，然后接下来让策略自己继续玩，这样可以得到期望奖励$Q_\pi(s,a)$

3. Q函数的两种写法

   - 输入是状态和动作，输出是一个标量

     - 这种Q函数既适合连续动作，又适用于离散动作

   - 输入是一个状态，输出是多个值，这种Q函数只适用于离散动作

     - 假设动作是离散的，Q函数输出的n个值就代表n种动作的Q值

     ![image-20260224220232424](assets/image-20260224220232424.png)

4. 虽然q函数只能评估某一个策略$\pi$的好坏，但只要有了Q函数，我们就可以进行强化学习，从而决定采取哪一个动作，从而进行策略改进
   - 假设一开始有一个演员，也许一开始很差，即便是随机的也没有关系
   - 初始演员开始和环境交互，收集数据，接下来学习Q值，去衡量pi在某一个状态下强制采取一个动作，接下来会得到的期望奖励
     - 用时序差分和蒙特卡洛方法都可以
   - 当学习出一个Q函数后就可以找一个新的策略，新的策略比原本的策略要好（稍后会定义什么是好）
   - 用新的策略代替老的策略，重复上面的步骤，策略会越来越好

5. 什么是好的策略：对于所有哦可能得状态s而言，$V_{\pi'}(s)\geq V_{\pi}(s)$
   - 也就是所有状态下，新策略严格占优

6. 如何从Q函数生成策略：
   - $\pi'(s)  = \underset{a}{argmax}Q_\pi(s,a)$
   - 假设我们学习出了$\pi$的Q函数，如果我们在所有状态下，将可能得a一一代入Q函数，看看哪个a可以让Q函数的值最大，这个动作就是$\pi'$会采取的动作

7. 注意Q函数计算和正常策略的区别：给定状态s和策略pi并不一定会执行动作a，但是计算q函数需要给定某个状态强制采取动作a之后由策略交互，这才是Q函数的定义

8. 老策略和新策略的关系：老策略在状态s采取动作a与新策略不一定是一样大的，新策略采取的动作会让其得到较大的奖励，所以新策略是用Q函数推出来的
   - 注意到，策略pi不是策略梯度中那样有个网络决定的，而是由Q函数通过上面的函数生成的
   - 新策略对于老策略一定严格占优

### 目标网络

1. 目标网络是深度Q网络一定会用到的技巧

2. 目标网络：在学习Q函数时也会用到时序差分的方法

   - 问题：假设收集到一个数据，比如状态$s_t$采取动作$a_t$后，得到奖励$r_t$，进入状态$s_{t+1}$，根据Q函数

     - $Q_\pi(s_t,a_t) = r_t + Q_{\pi}(s_{t+1}, \pi(s_{t+1}))$
     - 我们希望Q函数输入$(s_t,a_t)$y与输入$(s_{t+1}, \pi(s_{t+1})$得到的结果之间的插入是$r_t$
     - 但是这实际上不好学习，假设这是回归问题，损失函数时均方误差，$Q_\pi(s_t,a_t)$是网络的输出，$r_t + Q_{\pi}(s_{t+1}, \pi(s_{t+1}))$是目标，随着Q网络更新，目标是会变动的
       - 当然如果要实现这样的训练也可以，在做反向传播的时候$Q_\pi$是会变动的，把两个更新的结果加在一起（因为是同一个模型$Q_\pi$，所以两个更新的结果会加在一起）
       - $L(\theta) = \left( Q_\theta(s_t,a_t) - \big( r_t + Q_\theta(s_{t+1}, \pi(s_{t+1})) \big) \right)^2$
       - $\delta = Q_\theta(s_t,a_t) - r_t - Q_\theta(s_{t+1},\pi(s_{t+1}))$
       - 对损失求导：$\nabla_\theta L = 2\delta \left( \nabla_\theta Q_\theta(s_t,a_t) - \nabla_\theta Q_\theta(s_{t+1},\pi(s_{t+1})) \right)$
       - 发现同一个网络的梯度被加（实际上是方向相反的）在了一起
       - 因此会导致训练不稳定

     

     ![image-20260225224652482](assets/image-20260225224652482.png)

   - 方法：把其中一个网络给固定住，训练时只更新左边Q网络的参数，而右边的Q网络参数固定，因为右边负责产生目标，所以被称为目标网络

     - 当左边Q网络更新多次后，使用更新后的Q网络替换目标网络（将参数覆盖）

### 探索

1. 问题：当使用Q函数的时候，策略完全取决于Q函数，给定一个状态，穷举所有动作，采取让Q值最大的动作
   - $a = \underset{a}{argmax}Q_\pi(s,a)$
   - 使用Q函数来决定动作与使用策略梯度不一样，策略梯度的输出是随机的（输出的是动作的分布，根据分布采样）
   - Q函数中动作总是固定的，这不是一个好的采集方式
   - 要估测某一个状态采取某一个动作会得到的Q值吗，一定要在该状态采样到该动作
     - 对于表格，没有探索过，无法计算Q值
     - 对于网络问题稍小，因为有泛化能力，但仍然是个问题
   - 举个例子：假设要估计某种状态s，可以采取动作$a_1,a_2,a_3$
     - 假设三个动作都没有采样过，估计出来的Q值可能都是初始值
     - ![image-20260225230901042](assets/image-20260225230901042.png)
     - 如果才采取了动作2，它得到的值是正的奖励，Q值就会永远比其他动作，其他动作再也不会被采样到了
     - ![image-20260225231109229](assets/image-20260225231109229.png)
2. 方案：探索
   - epsilon-greedy
     - ![image-20260225231205304](assets/image-20260225231205304.png)
     - epsilon通常是一个很小的值，可能是0.1
     - 通常会随着时间衰减
   - 玻尔兹曼探索
     - 假设对于任意的s,a,$Q(s,a)\geq 0$
     - a被选中的概率$e^{Q(s,a/T)}$呈正比
     - ![image-20260225231459790](assets/image-20260225231459790.png)
     -  探索：如果T很大，动作几乎以等概率选择
     - 利用：如果T很小，Q值大的动作更容易被选中，T趋于0，只选择最优动作
     - 直观理解：可以简化为sigmoid
       - $p_1 = \frac{e^{z_1/T}}{e^{z_1/T}+e^{z_2/T}} =  \frac{1}{1 + e^{(z_2 - z_1)/T}}$
       - 如果T越大，z2和z1差距被缩小，如果z2-z1>0 p1会变小，p2会变大，因此所有选项之间概率会相近，趋于随机选择

### 经验回放

1. 经验回放：构建一个回放缓冲区（replay buffer），对于现有策略pi和环境交互，将收集的数据放到一个缓冲区中，回放缓冲区又被称为回放内存（replay memory）

   - 一笔数据：在某个状态$s_t$，采取某一个动作$a_t$，得到奖励$r_t$，进入状态$s_{t+1}$

   - 回放缓冲区中的经验可能来自于不同策略，只有装满时，才会把旧数据丢掉

     - 通常不会让某一个策略直接将缓冲区装满
     - 策略变成了异策略算法，本来Q是要观察$\pi$的经验，但实际上存储在回放缓冲区里的这些经验不完全来自于$\pi$，有些策略是其他策略留存在缓冲区中的

     ![image-20260302145511949](assets/image-20260302145511949.png)

2. 回放缓冲区的使用方法：迭代训练Q函数，每次迭代里边，从缓冲区中随机挑选一个批量出来，进行学习

3. 经验回放的作用：

   - 强化学习智能体与环境的交互产生的经验数据是 **顺序相关的**，如连续时间步的状态–动作–奖励–下一状态，这与监督学习中 i.i.d.（独立同分布）数据的假设相悖。直接使用顺序样本训练，会导致训练不稳定、收敛缓慢甚至失败
     - 经验回放机制通过把所有经验存入一个缓冲区，然后 **随机重抽样训练**，实现：
       - **打破时间相关性（decorrelation）**——使训练样本在统计上更接近 i.i.d.；
       - **多次利用同一条经验**——显著提高数据利用效率。 
- 强化学习的目标是最大化长期回报，这依赖于对期望值进行估计。不同的采样策略只是改变了样本分布权重，但仍然可以通过适当的权重修正（比如重要性采样）恢复对真实目标的无偏估计

### 深度Q网络

1. 深度Q网络：

   - 初始化两个网络，q网络Q和目标网络$\hat{Q}$，一开始两个网络是一样的$\hat{Q} = Q$

   - 每个回合中，使用演员和环境交互

     - 在某个状态$s_t$，采取某一个动作$a_t$，得到奖励$r_t$，进入状态$s_{t+1}$
     - $a_t$动作由Q函数结合探索机制确定
     - 完成一笔数据收集$(s_t,a_t,r_t,s_{t+1})$
     - 放入经验缓冲区
       - 经验缓冲区满的话讲旧数据丢弃

   - 在经验缓冲区中使用特定采样方法，采样批量经验

   - 使用目标网络计算目标$y = r_i+\underset{a}{max}\hat{Q}(s_{i+1},a)$

     - 其中a是让$\hat{Q}$最大的动作值

   - 更新Q值：回归问题

   - 更新一定次数C后，令$\hat{Q} = Q$

     ![image-20260302151810450](assets/image-20260302151810450.png)

2. **深度 Q 网络（DQN）与 Q 学习的主要区别：**

   - 函数表示方式不同

     - **Q 学习**：

       使用**表格（Q-table）**存储每个状态-动作对的价值。

       适用于状态空间和动作空间较小的情况。

     - **深度 Q 网络（DQN）**：

       使用**深度神经网络**来近似动作价值函数 Q(s, a)。

       适用于状态空间较大甚至连续的情况（如图像输入）

   - 训练数据使用方式不同

     - **Q 学习**：通常使用当前交互得到的下一个状态样本直接更新（在线更新）。

     - **DQN**：使用**经验回放（Experience Replay）**机制：

       - 将历史交互数据存入经验池
       - 随机采样小批量数据进行训练
       - 打破数据相关性，提高训练稳定性


## 深度Q网络进阶技巧

### 双深度Q网络 DDQN

1. 问题：Q值被高估了

   - 实际中Q值往往是被高估的：图中有 4 个不同小游戏。横轴表示训练轮次，红色曲线表示 Q 函数在不同状态下估计得到的平均 Q 值。随着训练进行，策略不断改进，因此在同一状态下获得的期望奖励逐渐增大，Q 值整体呈上升趋势。不过这些值是深度 Q 网络的估计值。通过大量采样（例如运行游戏 100 万次）计算得到的真实累计奖励往往比估计值小，因此会出现 Q 值过估计的问题。DDQN 的方法可以有效减轻这种过估计，使估计值更接近真实值。

     ![image-20260304171056202](assets/image-20260304171056202.png)

   - DDQN的优势：图 7.1 中蓝色实线表示 DDQN 的 Q 网络估计得到的 Q 值，蓝色虚线表示真实 Q 值，两者较为接近。相比普通 DQN，DDQN 在三种情况下得到的真实 Q 值更高，说明其学习到的策略更好，因此获得的实际奖励也更高。普通深度 Q 网络虽然会高估可能获得的奖励，但实际得到的奖励往往较低。

2. 为什么Q值会被高估

   - 在训练过程中，如公式 (7.1) 所示，我们希望 **当前估计值（左式）尽量接近目标值（右式）**。但是在实际计算目标值时，很容易出现 **目标值被高估的问题**。原因是：在计算目标值时，我们实际上是在所有动作 a 中选择 **最大的 Q 值**，然后将其作为目标的一部分。

   ​	![image-20260304171952753](assets/image-20260304171952753.png)

   - 假设现在有 **4 个动作**：这些动作真实的 **Q 值其实差不多**得到的 **奖励也差不多**。但是由于神经网络在估计 Q 值时 **存在误差**，某些动作的 Q 值可能会被 **偶然高估**。

     - 情况一（图 7.2a）

       - 第 1 个动作的 Q 值被高估，绿色部分表示被高估的量

       - 智能体在选择动作时，会认为这个动作的 Q 值最大，因此：

         - 选择这个动作

         - 用这个被高估的 Q 值再加上$ r_t$ 作为目标值

     - 情况二（图 7.2b）
       - 如果 **第 4 个动作**被高估：智能体就会选择第 4 个动作并用这个高估的 Q 值加上 $r_t$ 作为目标值
         - ![image-20260304172138890](assets/image-20260304172138890.png)
     - 这种情况下，因为总是选择了误差被高估的动作，并将奖励累加，因此目标值总是太大，学习并估计的Q值就会偏大

3. 方法：将选动作的Q函数和计算值的Q函数分开

   - 原本做法：

     - 穷举动作a，选择使Q值最大的动作a，然后加上在Q值上加上$r_t$
     - 使用 **目标网络** Q‘同时完成两件事：
       1. 选择最大 Q 值的动作
       2. 计算该动作的 Q 值

   - DDQN：

     - Q在线网络决定哪一个动作的Q值最大
     - Q'目标网络计算选出动作的Q值

   - 为什么可以：如果Q网络高估了动作a的Q值并选择了a，但是Q'没有高估Q值，算出来的值还是正常的

     - 只有两个网络同时高估的时候才会出问题，高估的概率被大大减小了

   - DDQN实现

     - 原本：targetnet输入下一个状态，选出最大的Q值
     - DDQN：
       - 改为使用onlinenet先输入下一个状态，计算Q值，选出最大Q的下一个动作
       - 使用targetnet输入下一个状态，使用onlinenet选的动作的Q值

   - DDQN 相比 DQN 的优点

     - 改动非常小，几乎不增加计算量
     - **不需要增加新的网络**：原本的 DQN 就已经有两个网络（在线网络 + 目标网络）DDQN 只是把选动作,算 Q 值这两步 分别交给两个网络。

     - ![image-20260304172821608](assets/image-20260304172821608.png)

### 竞争深度Q网络

1. 竞争深度Q网络：相比于DQN修改了网络架构

   - Q网络输出状态，直接输出每一个动作的Q值

   - 竞争深度Q网络分成两条路径

     - 第一条路径输出一个标量$V(s)$

     - 第二条输出一个向量$A(s,a)$，每个动作有一个值

     - 将两个值相加，得到Q值

       ![image-20260304200806313](assets/image-20260304200806313.png)

2. 演示：假设状态是离散的（实际上状态不是离散的），有4个不同的状态，3个不同的动作，因此Q可以看成一个Q表格

   - $Q(s,a) = V(s)+A(s,a)$
   - 实际上是在通过修改V和A来修改最终的Q值，引入V的好处是，V能同时修改到没有被采样的动作，这更好的利用了数据使Q的更新更高效

   - ![image-20260304200933165](assets/image-20260304200933165.png)

   - 退化问题：智能体可能最后学到V是0，A是Q，模型退化为了DQN

     - 给A施加约束，让A的更新更麻烦，从而让网络倾向于更细V

     - 常见的约束

       - $\sum_a A(s,a) = 0$

         - 在同一个状态 s 下，**所有动作的 Advantage 的和必须为 0**。
         - 强迫网络在提升所有动作优势的情况下选择更新V，而不是更新所有A

       - 如果满足这个条件，那么V(s)就可以理解为：该状态下所有 Q(s,a) 的平均值。因为：Q(s,a) = V(s) + A(s,a)

         - **V(s)** 表示状态的平均价值
         - **A(s,a)** 表示某个动作相对于平均水平的优势

       - 实现

         - 如下图，给$A(s,a)$施加一个约束，在A和V加起来前，先进行归一化（去均值化）

           - 计算均值，减去均值

         - 将每个元素加上V得到Q值

           - 将归一化写入网络，归一化没有参数，可以直接放入共同训练

           - 反向传播变化如下

             - $Q(s,a)=V(s)+A(s,a)-\frac{1}{N}\sum_{k}A(s,k)$
             - $\frac{\partial Q(s,a)}{\partial A(s,i)} = \begin{cases} 1 - \frac{1}{N}, & i = a \\ -\frac{1}{N}, & i \ne a \end{cases}$

           - 以这个图中为例

             - $A=[A_1,A_2,A_3]$

             - $Q_1=V+A_1-\frac{A_1+A_2+A_3}{3}$

             - 

             - | **参数** | **梯度**            |
               | -------- | ------------------- |
               | A₁       | $1-\frac13=\frac23$ |
               | A₂       | $-\frac13$          |
               | A₃       | $-\frac13$          |

             - 更新一个动作会影响所有动作的 Advantage:梯度更新会自动满足：$\sum_a A(s,a)=0$.因为$\sum_a \frac{\partial Q}{\partial A(s,a)} =0$.所以训练过程中Advantage 会自动保持零均值。

           - 这样会让A有比较大的约束，因为改一个动作其它动作也会被影响

           - 实现让网络更新，因为V只需要V(s)+1梯度会更稳定

         - ![image-20260304202043682](assets/image-20260304202043682.png)

### 优先经验回放（Prioritized Experience Replay， PER）

1. 问题：原始算法从缓冲区中均匀采样数据，但是实际上有些数据更重要
   - 直觉：如果有些数据时序差分误差特别大（网络输出和目标之间的差距），代表这些数据是不好训练的
     - 时序差分误差$\delta_i = Q(s_i,a_i) - (r_i + \gamma \max Q(s'_i,a))$
   - 这些数据应当给予优先权，不好训练的数据基于优先权(Priority)，在训练过程中更大概率被采样到
   - 实际PER实现时不仅改变了采样的过程还改变了更新参数的方法

2. PER算法：
   - $p_i = (|\delta_i| + \epsilon)^\alpha$
     - 其中epsilon防止优先级为0
     - alpha控制优先级强度，alpha为0退化成均匀采样，alpha越大越偏向TD error大的样本
   - 采样概率：$P(i) = \frac{p_i}{\sum_k p_k}$
     - TD error 大 → priority 大
     - priority 大 → 采样概率高
   - 改变采样分布会产生bias，导致网络更新不再是无偏估计
     - 引入重要性采样权重：$w_i = (N \cdot P(i))^{-\beta}$
     - beta是误差修正强度
       - beta = 0不修正， beta=1完全修正
     - 修正loss更新
       - 普通 DQN loss：$L = (\delta)^2$
       - PER 修改为：$L = w_i \cdot (\delta_i)^2$
         - $loss = (td_error ** 2) * weights$

3. 实现流程

   - 存储transition：初始化priority新样本 priority = 当前最大 priority
   - 根据 priority 计算采样概率batch$P(i) = \frac{p_i}{\sum p_k}$
   - 计算 importance sampling weight$w_i = (N P(i))^{-\beta}$并归一化$w_i = (N P(i))^{-\beta}$
   - 计算TD error
   - 更新网络
   - 更新priority$p_i = (|\delta_i| + \epsilon)^\alpha$

### 在蒙特卡洛方法和时序差分方法中取得平衡

1. 多步方法：在蒙特卡洛和时序差分学习中取得平衡的方法

   - 在时序差分的基础上保存N步的数据

2. 多步回报：估计的是状态$s_{t+N+1}$

   - $G_t^{(N)} = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + ... + \gamma^N Q(s_{t+N}, a_{t+N})$
     - 前 N 步使用真实奖励
     - 最后一步使用 Q 网络对状态$s_{t+N}$估计值

   - 折扣求和t到t+N的N+1个时间步的奖励的折现
   - 希望$Q(s_t,a_t)$和$G_t$越接近越好

3. 实现

   - ![image-20260304231751088](assets/image-20260304231751088.png)

   - 采样

     - 在$s_t$ 执行动作 $a_t$

     - 得到奖励$r_t$

     - 进入状态$s_{t+1}$

     - 然后持续记录：$(s_t,a_t,r_t,s_{t+1})$直到第 **N 步**：

       - 在 $s_{t+N}$ 执行动作$ a_{t+N}$

       - 得到奖励 $r_{t+N}$

       - 转移到状态 $s_{t+N+1}$

     - 更新 Q 网络
       - 在 **N-step 方法** 中目标是 **N 步后的状态** $s_{t+N+1}$
       - $G_t^{(N)} = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + ... + \gamma^{N-1} r_{t+N-1} + \gamma^N \hat Q(s_{t+N}, a_{t+N})$

4. 优劣

   - 优点：
   - 采样多步，减少了估测部分误差造成的影响
   - 坏处
   - r有很多项，把N项的r累计起来方差非常大
     - N作为一个超参数来调整

### 噪声网络

1. 探索：$\epsilon$贪心是一种在动作空间上添加噪声来进行探索的方式

   - 我们可以更聪明的添加噪声
2. 噪声网络：在参数空间上添加噪声

   - 在每一个 **episode（回合）开始时**，当智能体与环境进行交互时，智能体使用 **Q 函数来选择动作**。

   - 在噪声网络中，我们会在网络的 **每一个参数上加入高斯噪声（Gaussian Noise）**

     - $\theta = \theta + \epsilon$

     - $\theta$ ：原始网络参数
     - $\epsilon$ ：高斯噪声

   - 这样就会把原来的 **Q 函数**,变成一个 **带噪声的 Q 函数**：$\tilde{Q}$
     - 目标网络用了hat，噪声网络就用tilde
   - 使用方法：使用噪声网络选择执行动作
     - ![image-20260304232355940](assets/image-20260304232355940.png)
3. openai与DeepMind的噪声网络算法

   - OpenAI：对每一个参数、权重直接加入高斯噪声
   - DeepMind：噪声由一组参数控制，网络可以自己决定噪声要加多大
4. 噪音在一回合中不能改变：噪声在于环境交互之前就要采样，接下来噪声要一直固定知道游戏结束才重新采样

   - 原因：NoisyNet 与 **ε-greedy** 等传统探索方式有本质区别
     - 在 ε-greedy 中即使在相同状态下，智能体的动作也可能不同原因是动作是通过 **随机采样** 选择的
       - 在动作中加噪音
       - 有时按照Q执行有时随机执行
     - 但在现实世界中，一个合理的策略应该是**相同状态 → 相同决策**
       - 在网络中加噪音不会产生这个情况，因为网络参数虽然被扰动，但是在回合内是固定的
       - 看到类似的状态会采取类似的动作
   - 依赖状态的搜索（state-dependent exploration）
     - 探索与状态有关
     - 在参数中进行改变会系统性影响一回合中的整体行为
5. 注意：**实际应用中并非如此**
  - 更新噪声：
    - Get_action前对onlinenet重置一次
    - 更新参数前对online和target都要重置
  - 部分噪声注入：只对**顶层网络**（对于多层只用最后2-3层）注入噪声否则容易无法收敛
    - 实验中注意到uniform_重置反而在多层全部改为噪声网络时更好收敛，因子化反而没有办法收敛
  - Sigma_init一般0.5
    - 对于高维稀疏奖励可以放大
    - 低位密集奖励可以减小
  - 建议增大隐藏层数量（噪声注入减少了有效神经元容量）
    - 增大批次大小，减少buffer容量
    - 降低学习率

### 分布式Q函数

1. 问题：

   - 因为环境具有随机性，所以在某一个状态采取某一个动作的时候，我们把所有的奖励在游戏结束的时候进行统计，得到的是一个分布

     - 同一个动作可以产生不同的结果
     - 回报本身就是随机变量

   - Q函数是累计奖励的期望值，所以算出来的Q值是一个Q值分布的一个期望值

   - 但是不同的分布可以由相同的期望值，如下图

     ![image-20260309152047712](assets/image-20260309152047712.png)

   - 我们用Q值来代替了整个分布，可能会丢失一些信息从而对整个分布进行良好的建模

2. 分布式Q函数：对分布进行建模

3. 分布式Q函数实现：在原本的Q函数中，假设只能采取a1、2、3三种动作，输入一个状态输出三个值，每个值代表一个动作的Q值

   - 这个Q函数的输出实际上就是分布

     ![image-20260309152338062](assets/image-20260309152338062.png)

   - 回报范围离散化：
     - 假设每一个动作的奖励空间可以拆成5个长条l，Q函数的输出就是要预测在某一个状态采取某一个动作得到的奖励落在某一个长条内的概率
     - 将动作的奖励空间离散化成区间
     - 对于每个动作，网络输出一个每个区间的概率，且概率和满足为1
     - 上图中绿色代表一个动作的分布，红色代表一个，蓝色代表一个
     - 实际测试的时候选择平均值最大的动作执行
   - 除了选择平均值最大的动作，还可以对分布进一步建模
     - 考虑动作的分布，如果分布的方差很大，代表虽然平均不错但是风险很高，可以训练一个网络来规避风险
     - 在两个动作平均值相差不多的情况下，可以选择一个风险比较小的来执行
   - 分布式Q函数不容易高估奖励：
     - 概率分布的约束减少了极端预测值：学习了整个回报的分布而不是单一Q值，在对分布进行离散的时候我们会设定范围，丢弃极端预测值，因此很少高估奖励，而是低估奖励
     - 分布也能表示方差信息，因此理论上预测会更好，这减少了argmax操作带来的系统性误差

## 针对连续动作的深度Q网络

1. 深度Q网络V.S.策略梯度：
   - 优势：深度Q网络更稳定，策略梯度比较不稳定，玩大部分游戏不能使用策略梯度
     - 在没有PPO之前，策略梯度用处很小：最早DeepMind用深度强化学习玩雅达利的游戏用的就是深度Q网络
     - 原因：深度Q网络只要能拿到Q值就能找到一个比较好的策略，也就是只要能估计Q值就能改进策略
       - 估计Q值是一个回归问题，比较容易
       - 观察MSE下降就能知道模型学习的好坏
   - 劣势：很难处理连续动作，驾驶（方向盘角度），机器人动作（关节角度）都是连续的，其动作是一个连续的向量
     - 使用深度Q网络很难处理Q函数得到后的$a = \underset{a}{argmax}Q(s,a)$动作选择问题
     - 离散问题中可以代入每个动作，但是连续的话无法穷举
2. 方案1：对动作进行采样
   - 采样出有限的N个可能动作，把他们代入Q函数比较
   - 并不会太低效，因为采用了GPU可以并行计算
   - 不是很精准，没有办法进行太多采样，估计的Q值可能不精准
3. 方案2：路径衍生策略梯度——梯度提升
   - 优化问题：最大话目标函数，把a当做参数，找一组a去最大化Q值，然后用梯度提升更新a
   - 全局最大值：不一定能找到最优结果
   - 运算量很大：要迭更新a，训练强化学习网络本身很花时间，再要在每一次采用动作的时候训练网络，运算量非常大
4. 方案3：设计网络架构
   - 设计特别的网络架构尤其是Q函数来让解决argmax变得容易
   - ![image-20260309160523464](assets/image-20260309160523464.png)
   - 通常输入状态s是图像，可以使用向量或者矩阵表示
   - Q函数输出向量$\mu(s), \Sigma(s), V(s)$
   - 输入a和上面的输出向量互相作用
     - ![image-20260309160654837](assets/image-20260309160654837.png)
     - a是一个向量，元素是连续的
       - 假设a和mu是列向量
     - $\Sigma(s)$是一个正定矩阵，因为$\Sigma(s) = LL^T$，L是一个下三角矩阵
       - 设 A 为 n 阶矩阵，如果对于任意非零的 n 维向量 x，都有$x^T A x > 0 $则称矩阵 A 为**正定矩阵**。
     - 最终得出来是一个标量，且一定是正的
   - 如何找到最大化a的Q：让第一项越小越好，实际上就是a=mu
   - 局限：函数不能随便设置
5. 方案4：不适用深度Q网络
   - 使用基于策略的方法PPO和基于价值的方法深度Q网络结合在一起
   - 得到演员评论家算法

## 演员-评论家算法

1. 强化学习的两大类方法

   - 策略梯度：典型算法如Reinforce

     - 问题：每次需要采集一个完整的轨迹，并计算轨迹上的回报

       - 采样方差大

       - 学习效率低

   - 价值函数：典型算法如Q_learning、DQN

     - 问题：
       - 只能用于 **离散动作**
       - 不直接优化策略

2. 方案：借鉴时序差分的思想，使用动态规划方法来提高采样效率
   - 从状态s开始的总回报可以通过当前动作的即时奖励$r(s,a,s')$和下一个状态$s'$的值函数来近似估计

3. 演员-评论家算法：结合策略梯度和时序差分学习的强化学习方法
   - 演员：策略函数$\pi_\theta (a|s)$
     - 学习一个策略以得到尽可能高的回报
   - 评论员：价值函数$V_\pi(s)$
     - 对当前策略的值函数进行估计，评估演员的好坏
   - 借助价值函数，演员-评论员算法可以进行单步参数更新，不需要等到回合结束才进行更新
     - actor更新最大化$\nabla J(\theta) = E[ \nabla_\theta \log \pi(a|s) A(s,a) ]$
       - 其中优势函数可以用TD近似$A = r + \gamma V(s') - V(s)$
         - $target = r_t + \gamma V(s_{t+1})$
         - $A = target - V(s_t)$
     - critic更新：最小化$(V(s_t)-target)^2$

4. 演员评论员算法变体：
   - 优势演员评论家算法：A2C
   - 异步优势演员评论家算法：A3C

5. 策略梯度回顾：

   ![img_v3_02v7_fd2ffe51-ab31-4609-81f0-e955c89dd7fg](assets/img_v3_02v7_fd2ffe51-ab31-4609-81f0-e955c89dd7fg.jpg)

   - 策略梯度中

     - 表示我们首先通过智能体与环境的交互，可以计算出在某一个状态 s 采取某一个动作 a 的概率 $p_\theta(a_t|s_t)$
     - 接下来，我们计算在某一个状态 s 采取某一个动作 a 之后直到游戏结束的累积奖励。
     - $\sum_{t'=t}^{T_n} \gamma^{t'-t} r_{t'}^n$ 表示我们把从时间 t 到时间 T 的奖励相加，并且在前面乘一个折扣因子，通常将折扣因子设置为 0.9 或 0.99 等数值，与此同时也会减去一个基线值 b，减去值 b 的目的是希望括号里面这一项是有正有负的。
       - 如果括号里面这一项是正的，我们就要增大在这个状态采取这个动作的概率
       - 如果括号里面是负的，我们就要减小在这个状态采取这个动作的概率

   - 用G表示累计奖励，G非常不稳定，因为交互的过程本身有随机性

     - 在某一状态s采取动作a时计算得到的累计奖励，每次结果是不同的
     - G是一个随机变量，对于同样的状态s和同样的动作a，G可能有一个固定的分布
     - 但由于通过采样收集数据，因此将某一个状态s采取动作a后知到游戏结束的奖励称为了G

   - 如果把G想象成一个随机变量

     - 实际上是在对G做采样，用采样的结果更新参数

     - G本身有潜在的固定分布，但是方差可能会非常大，因此收集到的G结果可能会非常不一样

     - 如果可以采样足够多次那么没有问题

     - 如果非常小的采样导致采样道德数据方差很大，结果会很差

       ![img_v3_02v7_c3d08c48-4f5e-4be9-b932-ac49e136e5cg](assets/img_v3_02v7_c3d08c48-4f5e-4be9-b932-ac49e136e5cg.jpg)

6. 深度Q网络回顾

   - 我们不能让整个训练过程变得稳定，能不能直接估计随机变量G的期望

     - 直接用一个网络去估计状态s采取动作a时G的期望值，如果可行，那么在随后的训练中使用期望值代替采样的值，让训练变得更稳定

   - 怎样使用期望代替采样结果

     - 引入基于价值的方法，基于价值的方法就是深度Q网络

       - 两种评论员函数

         - 评论员$V_\pi(s)$：假设演员的策略是$\pi$，使用$\pi$和环境交互，当智能体看到状态s时，接下来的累计奖励的期望值是多少
           - 接受s输出一个标量
         - 评论员$Q_\pi(s,a)$：把s与a当做输入，表示状态s采取动作a，接下来用策略$\pi$与环境交互，累计奖励的期望值是多少
           - 接受s，给每个a分配一个值

         ![img_v3_02v7_9e07ca1d-0b7d-4c9b-a603-906b663f6b6g](assets/img_v3_02v7_9e07ca1d-0b7d-4c9b-a603-906b663f6b6g.jpg)

### 优势演员-评论员算法

1. 评论员Q函数：如下图所示，随机变量G的期望值正好是Q值定义，即$E(G_t^n) = Q_{\pi\theta}(s_t^n,a_t^n)$

   - Q函数的定义：在某一个状态s，采取某一个动作a，假设策略师$\pi$的情况下所能得到的累计奖励的期望值，即G的期望值
   - 将Q函数应用在这里，就将演员和评论员方法结合在了一起
   - 

   ![image-20260309164608116](assets/image-20260309164608116.png)



2. 评论员V函数：前面提到基线可以用一个神经网络来表示，常用的是价值函数$V_{\pi\theta}(s_t^n)$

   - 价值函数的定义：假设策略是$\pi$，其在某个状态s一直与环境交互直到游戏结束，期望奖励有多大
   - V函数没有涉及动作，V是Q的对所有动作的期望，因此Q-V是有正有负的

3. 演员评论家算法：将G替换成了A，将b替换成了V

   - 风险：如果估计两个网络Q、V，估计不准的风险变成了原来的两倍

   - 方案：只估计一个网络V，并利用V的值来表示Q的值，$Q_\pi(s_t^n, a_t^n)$可以写成$r_t^n+V_\pi(s^n_{t+1})$的期望值

     - ![image-20260309183927372](assets/image-20260309183927372.png)

     - 在状态s采取动作a，会得到奖励r，进入状态$s_{t+1}$
     - 但是会得到什么奖励r，进入什么状态$s_{t+1}$是有随机性的，所以取期望才会等于Q函数
     - 如果把期望值去掉就可以吧Q函数用$r_t^n+V_\pi(s^n_{t+1})$的值取代
       - ![image-20260309184149925](assets/image-20260309184149925.png)
     - 把期望值去掉的好处是不需要估计Q了，只需要估计V，但是同时会引入一个随机的参数r
     - r是随机变量但是相对于G是一个比较小的值，因为是某一步的奖励
     - G是未来所有步骤方差的和，所以方差比较大，这里相当于将比较大的方差替换成了一个比较小的方差r

   - 为什么可以去掉期望：原始的异步优势演员评论家算法的论文尝试了各种算法，最后发现这个比较好，其他方法不OK

   - 算法流程：

     - 演员和环境交互
     - 使用时序差分或者蒙特卡洛的方法先估计价值函数
     - 基于价值函数使用，更新$\pi$
       - ![image-20260309184656310](assets/image-20260309184656310.png)
     - 重复上面的步骤

### 路径衍生策略梯度

1. 深度Q网络解连续动作的一种特别方法，也可以看做一种演员评论家模型

2. 从深度Q网络来看的问题：在使用一个深度Q网络时，考虑连续向量会比较麻烦，没有通用的解决方案

3. 从对抗式生成网络的角度来看：
   - 学习一个判别器用于评估非常困难：

3. 重要性采样

   - 采样：

     - 对于一个随机变量，我们通常使用**概率密度函数**来刻画它的概率分布特性。具体来说，给定随机变量的某一个取值，可以通过概率密度函数计算该取值对应的概率（或密度）。
     - 反过来，我们也可以根据概率密度函数所提供的分布信息，生成随机变量的一个取值，这个过程称为**采样**。因此，从某种意义上说，采样是概率密度函数的一种逆向应用。
     - 需要注意的是，与根据概率密度函数计算样本点对应的概率值不同，采样过程通常没有那么直接，往往需要根据待采样分布的具体特点，选择合适的采样策略。

   - 函数采样：

     - 给定一个函数 f(x)，我们希望计算在分布 p(x) 下 f(x) 的期望值：$\mathbb{E}_{x \sim p}[f(x)] = \int f(x)p(x)\,dx$

     - 如果我们无法对分布 p(x) 进行解析积分，但可以从该分布中采样得到样本$x_i$，那么可以使用**蒙特卡洛方法**来近似计算期望值。

     - 具体做法是：

       1. 从分布 p(x) 中采样得到 n 个样本$ x_1, x_2, \dots, x_n$；
       2. 计算每个样本对应的函数值$ f(x_i)$；
       3. 取这些函数值的平均：$\mathbb{E}_{x \sim p}[f(x)] \approx \frac{1}{n}\sum_{i=1}^{n} f(x_i)$

       当样本数量 n 足够大时，根据大数定律，该样本均值将收敛到真实的期望值。

   - 重要性采样

     - 问题：假设我们不能从分布p采样，只能从另外一个分布q采样数据x，q可以是任何分布
     - 我们需要对$\mathbb{E}_{x \sim p}[f(x)] \approx \frac{1}{n}\sum_{i=1}^{n} f(x_i)$做一个修正，因为x不是p采样出来的
     - $\mathbb{E}_{x \sim p}[f(x)] = \int f(x)\frac{p(x)}{q(x)}q(x)\,dx=\mathbb{E}_{x \sim q}[f(x)\frac{p(x)}{q(x)}]$
       - 这样通过从q中采样，计算$\mathbb{E}_{x \sim q}[f(x)\frac{p(x)}{q(x)}]$期望，就可以得到p采样x的期望值
       - 重要性权重：$\frac{p(x)}{q(x)}$，修正两个分布之间的差异
         - 其中，q(x) 可以是任意分布，唯一的限制是：当 q(x) = 0 时，必须同时满足 p(x) = 0，否则比值 \frac{p(x)}{q(x)} 就没有定义

     - 重要性采样的实际应用要求：q理论上是任意的，但是实际上p和q的差距不能太大
       - 式子$\mathbb{E}_{x \sim p}[f(x)] = \int f(x)\frac{p(x)}{q(x)}q(x)\,dx=\mathbb{E}_{x \sim q}[f(x)\frac{p(x)}{q(x)}]$虽然期望上成立
       - 但是方差是不相等的
         - ![image-20260303172038695](assets/image-20260303172038695.png)

       - 如下图：假设蓝线是p(x)的分布，绿线是q(x)的分布，红线是f(x)
         - 计算f(x)期望时，如果从p(x)采样，E(f(x))显然是负的，因为左边的概率高
         - 但是如果从q(x)采样，因为右边的概率高，E(f(x))期望是正的
         - 就算通过重要性权重进行了修正，如果采样次数不够多没有采样到左边，那么无论怎么乘权重都无法还原

       - ![image-20260303172139852](assets/image-20260303172139852.png)

2. PPO的优化约束

   - PPO的优化目标函数${J^\theta}^\prime(\theta)$，但是这个目标函数涉及到重要性采样，需要限制$p_\theta(a_t|s_t)，{p}_{\theta^\prime}(a_t|s_t)$不要相差太多

     - 注意是梯度提升，而KL散度越小越好，因此需要加上一个负号
   - 尽管 PPO 的优化目标涉及重要性采样，但它只使用上一轮策略 θ′ 采集的数据。由于在 PPO 的目标函数中加入了 KL 散度约束，使得行为策略 θ′ 与目标策略 θ 非常接近，因此可以认为两者近似为同一策略。基于这一点，PPO 通常被视为一种*同策略（on-policy）算法*。

3. TNPO**（Truncated Natural Policy Gradient）** 

   - 背景：策略梯度有几个严重的问题

     - 更新方向不合理
     - 步长很难选
     - 策略可能突然崩溃

   - 原因：

     - 参数空间是 **非欧几里得空间**，普通梯度并不是最优方向。
     - 于是 Amari 提出了 **Natural Gradient（自然梯度）**
     - 自然梯度通过 **Fisher Information Matrix (FIM)** 来修正梯度方向
     - 最早自然梯度是为神经网络训练而设计的，但是由于计算成本太高且SGD足够好较少采用
     - 之前提到过，策略梯度中参数更新大的目标实际上是为了策略发生变化，而策略实际上是网络的输出，是一个概率分布，在这里使用自然梯度直接在概率空间进行优化实际上更符合这里的场景

   - 自然梯度：保证策略更新前后不会变化太大

     - 普通策略梯度：$g = \nabla_\theta J(\theta)$

     - 自然梯度：$\tilde{g} = F^{-1} g$

       - 其中$F = E[\nabla_\theta \log \pi_\theta(a|s)\nabla_\theta \log \pi_\theta(a|s)^T]$

     - 自然梯度更新公式改为

       - $\theta_{new}=\theta+\alpha F^{-1}\nabla_\theta J(\theta)$

     - | **普通梯度** | **自然梯度**     |
       | ------------ | ---------------- |
       | 参数空间移动 | 概率分布空间移动 |
       | 更新不稳定   | 更新更稳定       |
       | 容易崩       | 保持策略接近     |

   - 问题：如果策略网络参数非常大，Fisher矩阵非常大，几乎无法求逆

     - TNPG使用共轭梯度来近似$F^{-1}g$
     - 求解$Fx = g$来求得$x = F^{-1}g$

   - 更新步骤

     - 采样 $trajectory (s_t, a_t, r_t)$
     - 计算 $advantage A(s,a)$
     - 计算策略梯度$ g = \nabla_\theta J(\theta)$
     - 计算 Fisher-vector product $Fv$
     - 使用 Conjugate Gradient 求解$ F^{-1} g$（即自然梯度方向）
     - 更新参数 $\theta \leftarrow \theta + \alpha d$
       - 这里的alpha由于进行过kl散度的截断，需要设置大一点，否则就训练不动了

     其中$ d \approx F^{-1} g$，通过共轭梯度迭代近似得到

5. KL散度的意义：KL散度约束了两个策略网络输出的动作概率分布之间的相似程度

   - 这与用范数约束网络参数$\theta$之间的相似度是不一样，参数的变化程度不等于输出分布的变化程度

### PPO策略的变种

1. PPO算法的两个变种
   - PPO1：近端策略优化惩罚(PPO-penalty)
   - PPO2：近端策略优化裁剪(PPO-clip)

#### 近端策略优化惩罚

1. 算法：

   - 初始化一个策略的参数$\theta^0$
   - 迭代
     - 用前一个训练迭代的得到的演员参数$\theta^k$与环境交互，采样到大量的状态-动作对
     - 估测$A^{\theta^k}(s_t,a_t)$
     - 使用PPO的优化公式
       - 可以用$\theta^k$得到的轨迹更新参数多次（不需要立马重新采样）

2. 自适应KL散度：

   - $\beta$该设置多少：动态调整
     - 设置一个可接受的KL散度最大值：假设优化完一次发现KL散度比最大值还大，说明约束不够强，真实的偏离还是很大，就增大$\beta$
     - 设置一个可接受的KL散度最小值：假设优化完一次发现KL散度比最小值还小，就说明约束太强，没有优化策略梯度只约束距离了，减小$\beta$

   - 实际含义：希望$p_\theta(a_t|s_t)，{p}_{\theta^\prime}(a_t|s_t)$不要相差太远
     - A大于0时，说明状态-动作对是好的，因此希望增大概率$p_\theta(a_t|s_t)$
       - 如果$\frac{p_\theta(a_t|s_t)}{{p}_{\theta^\prime}(a_t|s_t)}>1$，大过$1+\epsilon$就没有好处了，因此超过就需要停止训练
       - 如果$\frac{p_\theta(a_t|s_t)}{{p}_{\theta^\prime}(a_t|s_t)}<1-\epsilon$，那么无论如何我们希望概率越大越好，因此没有理由停止训练
     - A小于0时，说明状态-动作对是不好的，因此希望减小概率$p_\theta(a_t|s_t)$
       - 如果$\frac{p_\theta(a_t|s_t)}{{p}_{\theta^\prime}(a_t|s_t)}<1$，小于$1+\epsilon$就没有好处了，因此超过就需要停止训练
       - 如果$\frac{p_\theta(a_t|s_t)}{{p}_{\theta^\prime}(a_t|s_t)}>1+\epsilon$，那么无论如何我们希望概率越小越好，因此没有理由停止训练
